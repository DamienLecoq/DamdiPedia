## 🧠 Résumé rapide

Java est un langage très utilisé dans les systèmes backend et les applications d’entreprise. Il est connu pour sa stabilité, sa portabilité et son écosystème mature, mais peut être perçu comme verbeux.

---

## 🧠 Définition

Java est un langage de programmation orienté objet, compilé en bytecode et exécuté sur une machine virtuelle (JVM), permettant une portabilité entre systèmes.

---

## 📜 Histoire

- Créé en 1995 par Sun Microsystems

- Objectif : “Write once, run anywhere”

- Aujourd’hui maintenu par Oracle

- Très utilisé dans les grandes entreprises

---

## 🎯 Objectif

- Créer des applications robustes et maintenables

- Assurer la portabilité entre systèmes

- Fournir un cadre solide pour les applications complexes

---

## 🌍 Domaines d’utilisation

- Applications backend (API REST)

- Applications d’entreprise

- Android (historiquement)

- Systèmes distribués

---

## ⚙️ Fonctionnement

- Code compilé en bytecode

- Exécution sur la JVM

- Gestion automatique de la mémoire (Garbage Collector)

- Typage statique

---

## 💡 Concepts clés

- Orienté Objet (classes, héritage, polymorphisme)

- JVM (Java Virtual Machine)

- Garbage Collection

- Exceptions

- Packages

---

## 🧪 Exemple

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```

---

## 🧱 Structure / Architecture

- Classes

- Packages

- Interfaces

- Méthodes

---

## ⚡ Syntaxe et spécificités

- Typage statique

- Verbosité importante

- Utilisation de classes pour tout

- Point d’entrée : main()

---

## ✅ Avantages

- Portabilité (JVM)

- Écosystème très riche

- Performance correcte

- Typage strict (moins d’erreurs runtime)

---

## ❌ Inconvénients

- Verbosité

- Démarrage lent (JVM)

- Complexité pour les débutants

- Moins flexible que certains langages modernes

---

## ⚠️ Pièges courants

- Mauvaise gestion des exceptions

- Confusion entre == et equals()

- NullPointerException fréquentes

- Complexité excessive (over-engineering)

---

## ❗ À ne pas confondre

- JDK vs JRE vs JVM

- == vs equals()

- Compilation vs interprétation

---

## 🧠 Explication simplifiée

Java est comme une langue stricte pour parler à un ordinateur. Tu dois bien respecter les règles, mais en échange ton programme est plus fiable et fonctionne partout.

---

## 🧠 Explication avancée

Java repose sur une compilation en bytecode exécuté par la JVM. Ce modèle permet une abstraction du système d’exploitation. Le garbage collector gère automatiquement la mémoire, et le typage statique permet une vérification à la compilation.

---

## 🔥 Points critiques à retenir

- \[CRITIQUE\] Java utilise la JVM pour être portable

- \[IMPORTANT\] Langage fortement typé

- \[IMPORTANT\] Basé sur la programmation orientée objet

- \[PIÈGE\] == ne compare pas les objets correctement

- \[PIÈGE\] NullPointerException très fréquente
